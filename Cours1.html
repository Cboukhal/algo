Definition:
C'est une suite d'instrution prenant des données en entrée, réalisant un traitement et produit des donéées en sortie.

  • Un algo est indépendant de tout language de programmation car un programme n'est rien d'autre que l'implémentation d'un algo.
  • En algo on ne parle pas de code mais de pseudo-code.

user(algo) -> (language de programmation =traduction) -> machine(binaire = 01)
algo != (ARP = logiciel de programmation)

*Une instruction: c'est un ensemble d'élément exécutant une tache bien précise.

    Comme :
  • les variables
  • les constantes
  • les opérateurs
  • les boucles
  • les fonctions
  • les tableaux...

  3. Résultat

2. Les Variables

Définition

Une variable est un conténer, c'est comme une boite qui peut contenir divers éléments.Pour identifier cette boite, on lui cole une étiquette qui n'est rien d'autre que le nom de la variable. Le contenu de la boite est la valeur de la variable et le type d'élément contenu dans la boite est le type de la variable.

Déclaration d'une variable

La déclaration des variables se fait tout au début de l'algo, aprés la déclaration de l'algo même. Mais avant la 1er instruction concernant le traitement.
Pour déclarer une variable, on utilise le mot variable suivis du nom de la variableensuite de 2 points et du type de la variable.
Les var du même type peuvent être déclarer sur la même ligne

    exemple:
  • Variable prénom, nom: char
  • age : int
  • taille : float/double
  • des chaines de caractéres : char

Les types de variables

On attribut un type à une var selon son contenu.

    exemple:
  • Age => entier/int

  • 1. Booléen/Bool

  • C'est de type de donnée qui correspond à 2 alternatives: vrai==0 ou faux==1

  • 2. Entier/int

  • C'est de type de donnée qui correspond aux entiers naturels.

  • 3.Réel/float/double

  • C'est de type de donnée qui correspond aux nbrs décimaux.

  • 4.Chaine de caractéres/char

  • C'est de type de donnée qui correspond aux plusieurs caractéres.

Attention

Supposons notre var a:

  • "1" => a : char;
  • 1 => a : int;
  • 1,0 => : float/double;

Affectation d'une variable

C'est d'assigné une valeur à une var. Le propre d'une var étant de varier donc la notion d'égaliter pour l'affectation des var est inexistante en Algo.
Exemple: 32 -> Toto;
24 -> Toto; Toto = 24.

3. Les constantes

Définition

Une constante est un conténer qui contient une seul et unique valeur qui ne change jamais.
Exemple : 3.14 -> Pi;

Déclaration et affectation d'une constante

La déclaration de la constante et accompagnée de son affectation. Etant donné que la valeur de la constante ne changera pas, donc si on lui affecte sa valeur de la déclaration. Le type reste facultatif pour les constantes.
exemple : Pi ← 3,14;
Pi Réel ← 3,14;

Les boucles

Une boucle sert à répéter une instruction ou un bloc d'instruction autant de fois que nécessaire selon la || les condition(s) définis par le programmeur.

Tant que (while)

La boucle tant que test une condition et si la condition est vérifié elle éxécute les instructions, sinon elle ne fera rien.

    Synthaxe :
  • Tant que (condition) faire{
  • instruction
  • }Fintantque
    exemple :
  • n ← 0
  • Tant que n <=10 faire{
  • Ecrire n;
  • n ← n + 1;
  • } Fintant que;

  • 2
  • 3
  • 4
  • 5
  • 6
  • 7
  • 8
  • 9
  • 10

Répéter jusqu'à (Do while)

Elle éxecute d'abors les instructions et teste la condition par la suite, si la condition est vérifiée, elle éxecute les instructions à nouveau tant que la condition reste vraie. En définitive, la boucle répéter va éxecuter au moins une fois les instructions.

    Synthaxe :
  • Répéter {
  • instruction
  • }Jusqu'à(condition)
    exemple :
  • n ← 0
  • Répéter{
  • Ecrire ("j'adore les algos");
  • n ← n + 1;
  • } jusqu'à n == 5;
    résultat :
  • "j'adore les algos"
  • "j'adore les algos"
  • "j'adore les algos"
  • "j'adore les algos"
  • "j'adore les algos"

Tableau

Un tableau est un conteneur pouvant stocker plusieurs valeurs.

Definition

Supposons qu'on est besoin de 20 valeurs simultanéments. Il faudrait dans se cas créer 20 variables pour stocker ces valeurs. Volià pourquoi les tableaux sont importants dans la programmation. Parce qu'ils permettent de stocker plusieurs valeurs dans un et un seul tableau.
Un tableau peut contenir un ou plusieurs autres tableaux.
On accéde aux valeurs de notre tableau, grace à des indices qui vont de 0 à la longueur du tableau. Un tel tableau est appelé tableau indéxé. On peut aussi accéder aux valeurs d'un tableau grâce à leurs noms, dans se cas on appelle ce tableau associatif ou nominatif.

Déclaration d'un Tableau

Les tableaux tout comme les variables et les constantes se déclarent avant l'étape de traitement. On peut déclarer un tableau en précisant sa longueur à l'avance mais l'attribution de la taille du tableau et dynamique par défaut. C'est à dire quelle est automatiquement gérer par le systéme.

    exemple :
  1. Tableau nom_du_tableau(longueur du tableau): Type du tableau(facultatif)
  2. Tableau nom_du_tableau (dynamique)

Affectation

A la différence des variables et des constantes, l'affectation de valeur à un tableau se fait de maniére différente.
Car toute les valeurs vont être encadrés par des crochets[].

    exemple :
  1. nom_du_tableau ← [valeur1,valeur2,valeur3,...]
  2. nom_du_tableau = [valeur1,valeur2,valeur3,...]

Parcourir un tableau

Pour parcourirun tableau, on utilise une boucle pour accéder à l'ensemble des valeurs du tableau. On accéde à une valeur précise du tableau grâce au nom du tableau entre crochet l'indice de la valeur correspondante.
Dans tout language de programmation il existe des fonctions prédifinis pour déterminer ou renvoyer la longueur d'un tableau. Au cas où celui-ci serait déclarer dynamiquement.
Dans se cas si on connait notre indice de départ qui est généralement 0 et notre indice finale qui sera donné par la longueur du tableau, il est plus approprié d'utiliser la boucle pour(for), pour parcourir le tableau.

    exemple :
  • pour i de 0 jusqu'à longueurtab pas de 1 Faire{
  • Ecrire tab[i]
  • }Finpour

Modifier un Tableau

On peut modifier un tableau en y ajoutant des valeurs ou en supprimant, ou tout simplement modifier les valeurs déjà présente dans le tableau.

Ajouter une valeur au tableau :

Dans un tableau indexé on ajoute la valeur à la fin du tableau. Sinon on risque d'écraser les valeurs déjà présente.

    exemple :
  • taille ← longueur(tab)
  • tab[taille + 1] ← nouvelle_valeur;

Supprimer une valeur d'un tableau

Dans la plus part des languages de programmation il existe des fonctions prédéfinis qui permettent de supprimer une valeur d'un tableau.
Cependant en algorithmie nous allons utiliser le vide pour remplacer la valeur existante.

    exemple :
  • tab[7] = ""

Modifier une valeur

Pour modifier une valeur, on passe par son indice dans le tableau comme pour la suppression.

    exemple :
  • tab[7] ← nouvelle valeur

Tabbleau multidimensionnel

Definition

Un tableau est un conteneur qui peut contenir plusieurs valeurs, donc un tableau multidimensionnel n'est rien d'autre qu'un tableau de tableaux.

Déclaration

Le tableau multidimensionnel étant aussi un tableau, il se déclare aussi comme tout autre tableau.

    exemple :
  • tableau nom_du_tableau (longueur facultatif) : (type falcutatif)

Affectation

Comme pour les tableaux simples, l'affectation d'un tableau multidimensionnel se fait grâce ou à l'aide des crochets qui contiennent les valeurs.
Un tableau multidimensionnel étant un tableau de tableau donc on aura davantage de crochet.

    exemple :
  • tab ← [[prénom,nom,age], [prenom,nom,age], [prenom,nom,age]]
  • tab3 ← [[[prénom,nom,age], [prénom,nom,age]], [[prénom,nom,age],[prénom,nom,age]]]

Parcourir un tableau

tab ← [[8, 6, 2],[1, 3, 7],[6, 5, 2]]
Pour parcourir les tableaux, on aura besoin d'autant de boucle que de dimentions.
On a besoin d'une boucle pour parcourir est affiché les éléments d'un tableau simple.
Il est donc évident qu'on est besoin d'autant de boucles que de dimensions.
C'est à dire que j'aurai besoin de 2 boucles pour un tableau à 2 dimensions...

    exemple :
  • Pour i de 0 jusqu'à longueur(tab) faire{
  • Pour j de 0 jusqu'à longueur(tab(i)) faire{
  • Ecrire tab[i][j];
  • }finpour
  • }finpour

Modifier un TM

Tout comme un tableau simple, un TM peut être modifier en y ajoutant des valeurs, en y supprimant ou en les modifiants.

Ajouter des valeurs à un TM :

On ajoute une nouvelle valeur à la fin de notre tableau.

    exemple :
  • newtabs ← ["coucou","salut","bien"];
  • long ← longueur(tab);
  • tab[long+1] ← newtabs;

Suppression de valeur d'un TM

Bien que présente dans la programmation, en algorithmie, il n'existe pas de fonction prédéfinie pour la suppression du coup on remplace les valeurs par du vide.

    exemple :
  • newtabs ← ["","",""];

Modifier les valeurs d'un TM

Tout comme le tableau simple, on peut modifier les valeurs d'un TM en posant par ses indices.

    exemple :
  • tab[1][2] ← newsval

Les fonctions

Definition

Une fonction est un sous algorithme, c'est à dire peut contenir l'ensemble des éléments qu'on trouve dans algorithme. Elle porte un nom et posséde des paramétres(moyen de communication) et peut être appelé dans l'algorithme autant de fois que nécessaire.
Les variables, les tableaux ainsi que les constantes à l'intérieur d'une fonction deviennent des éléments locaux, c'est à dire qu'ils ne sont accéssible que dans la fonction.
La fonction est comme un raccourci qui permet d'éviter les longues répétition de lignes d'instructions. Elle a pour objectif de faciliter la lecture de l'algorithme et d'éviter la répétition. Il existe des fonction prédéfini, c'est à dire généré automatiquement par la langue ou le programme utilisé. Ou encoore par l'utilisateur.

Déclaration des fonctions

On déclare une fonction avec l'instruction fonction suivi du nom de la fonction et des parenthéses. A l'intérieur des parenthéses, on mettra si besoin les paramétres de la fonction. Si la fonction retourne un type bien défini on pourra rajouter dans se cas le type de la fonction précédé de 2points après les parenthéses.

    1. Synthaxe :
  • fonction nom_de_la_fonction()
  • Début{
  • instruction
  • }Fin
    exemple :
  • fonction bonjour()
  • Début{
  • Ecrire ("Bonjour, bonne journée");
  • }Fin
    résultat :
  • Bonjour, bonne journée
    2. Synthaxe :
  • fonction nom_de_la_fonction(val a : entier, ref b:chaine de caractère):Entier
  • Début{
  • instruction
  • retourne a;
  • }Fin
    exemple :
  • fonction addition(a,b):Entier
  • Début{
  • return a + b;
  • }Fin
    résultat :
  • a + b

Utilisation (appel à la fonction) :

On utilise les fonctions à l'intérieur de l'algorithme. On fait appel à la fonction par son nom suivi des parenthéses.
Si la fonction n'est pas vite et qu'elle retourne une valeur, il faudra stocker cette valeur afin de pouvoir l'afficher ou travailler avec.

    exemple :
  • bonjour();
  • c ← addition(5,3);

Fonctions récursives

Une fonction récursive est une fonction qui fait appel à elle même et à employer avec beaucoup de précaution. Car elle risque de boucler à l'infini et bloquer le systéme.

    exemple :
  • fonction bonjour()
  • Début{
  • Ecrire "Bonjour";
  • bonjour();
  • }Fin
    exemple :
  • fonction pgcd(a:entier,b:entier):Entier
  • Début{
  • si b = 0 alors{
  • retourne a;
  • }Finsi
  • retourne pgcd(b,a%b);
  • }Fin
